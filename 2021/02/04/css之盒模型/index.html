1.网络布局

本质：

  1. 首先利用css设置好盒子的大小，然后摆放盒子的位置。
  2. 最后把网页元素，eg：文字图片等，放入盒子内。

.2.盒子模型

Box Model : 把HTML页面中的布局元素看作是一个矩形的盒子，也就是一个承装内容的容器

2.summary：

  1. boxmodel包含元素内容，边框（border），内边距（padding），外边距（margin）
  2. model内的文字和图片等元素是内容区域
  3. 盒子的厚度就是盒子的border
  4. 盒子内容与边框的距离是内边距（类似单元格cellpadding）
  5. 盒子与盒子之间的距离是外边距（类似单元格cellspacing）

作用：控制盒子和盒子之间的距离

  • margin-left
  • margin-right
  • margin-top
  • margin-bottom

简写方法同padding一样：1/2/3/4.

8.块级盒子水平居中

  • 实现条件：
    • 盒子必须指定宽度
    • 左右外边距设置为auto

常见三种写法：

  • .class {width：200px; margin:0 auto;}

  • .class {width: 200px; margin: auto;}

  • .class {width: 200px; margin-left:auto; margin-right:auto;}

tip（注意区分）:

  • 文字居中与和盒子居中的区别：

    盒子内文字/行内/行内块元素 居中：text-align : center;

    盒子居中：eg：margin: 10px auto;

  • 插入图片和背景图片的区别：

  1. 插入图片： eg：产品展示类 移动位置只能靠盒模型 padding margin

    2.背景图片一般用于小图标背景 or 超大背景图片 通过 background-position改位置。

10.内外边距合并

  • 相邻块元素垂直外边距的合并

    • 当上下相邻的两个块元素相遇时，如果上面的元素有下外边距margin-bottom
    • 下面的元素有上外边距margin-top，则他们之间的垂直间距不是margin-bottom与margin-top之和
    • 取两个值中的较大者这种现象被称为相邻块元素垂直外边距的合并（也称外边距塌陷）。

  • 嵌套元素垂直外边距的合并（塌陷）

    • 对于两个嵌套关系的块元素，如果父元素没有上内边距paddin-top及边框border
    • 父元素的上外边距margin-top会与子元素的上外边距margin-top发生合并
    • 合并后的外边距为两者中的较大者！

解决方案：

  1. 可以为父元素定义上边框border-top
  2. 可以为父元素定义上内边距padding-top
  3. 可以为父元素添加overflow:hidden

11.盒子模型布局稳定性

按照 稳定性排序, 优先使用 宽度 （width）

其次 使用内边距（padding）

再次 外边距（margin）。

  • 原因：

    • margin 会有外边距合并

    • padding 会影响盒子大小，需要进行加减计算

    • width 没有问题。
