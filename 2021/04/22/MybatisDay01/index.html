MybatisDay01

1.框架

1.1什么是框架？

 它是我们软件开发中的一套解决方案，不同的框架解决的是不同的问题。

​ 使用框架的好处：

​ 框架封装了很多细节，使开发者可以使用极简的方式实现功能，可以大大提高开发效率。

1.2 三层架构

  • 表现层：用于展示数据
  • 业务层：处理业务需求
  • 持久层：和数据库交互

1.3 持久层技术解决方案

  • JDBC技术
  • Spring的JdbcTemplate：spring中对JDBC的简单封装
  • Apache的DButils：也是对Jdbc的简单封装

以上都不是框架，JDBC是规范

后两者都只是工具类

1.4 mybatis概述

  • mybatis 是一个由java编写的持久层框架
  • 封装了jdbc操作的很多细节，使开发者只需要关注sql语句本身，而无需关注注册驱动，创建链接等繁杂过程
  • 它使用了ORM思想实现了结果集的封装

ORM：

​ Object Relational Mappging 对象关系映射

​ 简单说：

​ 就是把数据库表和实体类及实体类的属性对应起来，让我们可以操作实体类就实现操作数据库表。

实体类中的属性 数据库表的字段
User user
userId id
userName user_name

1.4.1 mybatis入门

  • mybatis的环境搭建：
    1. 创建maven工程并导入坐标
    2. 创建实体类和dao的接口
    3. 创建mybatis的主配置文件： mybatis-config.xml
    4. 创建映射配置文件 UserMapper.xml —-用来写sql语句

  • 环境搭建的注意事项：

    1. 在Mybatis中它把持久层的接口操作名称和映射文件也叫做：Mapper

    2. 在IDEA中创建目录时，它和包是不一样的：

      包在创建时：io.zhj.dao 它是三级结构

      目录在创建时：io.zhj.dao 是一级目录

    3. mybatis的映射配置文件位置必须和dao接口的包结构相同

    4. 映射配置文件的mapper标签namespace属性的取值必须是dao接口的全限定类名

    5. 映射配置文件的操作配置（select），id属性的取值必须是dao接口的方法名。

    遵从3，4，5点后，开发中就无须再写dao的实现类
